# GitDOS — DOS в браузере, диск из GitHub

**Автор: Sementsul Maxim.** Сайт: https://sementsul.github.io/GitDOS/ · Репозиторий: https://github.com/sementsul/GitDOS

GitDOS — браузерный клиент **DOSBox** (движок **js-dos v8**, лежит локально), где диск C: — это
**GitHub-репозиторий**: чтение, запись и **автосохранение** прямо в git. Дополнительные репозитории
подключаются как папки `C:\DISKS\<буква>`. Интерфейс в стиле установщика Windows 98, RU/EN.

> Это веб-обёртка вокруг эмулятора. Сам DOSBox — GPL-2.0, js-dos — MIT (без изменений, в `vendor/`).
> Подробности и лицензии — `THIRD-PARTY.md`.

## Что умеет
- **Диск = репозиторий**: монтирование C: из публичного или приватного репо, доп. диски D:/E:…
- **Автосохранение** изменений в GitHub (пуш только новых/изменённых файлов по git-sha, удаления — тоже).
- **Большие файлы до 100 МБ** — через Git Data API (blob→tree→commit); чтение публичных файлов — через
  `raw.githubusercontent.com` (не тратит лимит API). Файлы >100 МБ — опционально через GitHub Release.
- **Профили подключения** (наборы «главный репо + доп. диски»), переключение/дублирование.
- **История диска** («Машина времени»): снимки прошлых версий (только чтение) и восстановление файла.
- **`.gitdos.json`** — манифест: имя диска, автозапуск программы (`run`), скорость CPU (`cycles`).
- **Лаунчер** — список программ (.EXE/.COM/.BAT), запуск по клику.
- **Выборочное монтирование** — грузить только нужные папки (быстрый старт больших дисков).
- **Загрузка файлов и .zip** (архив распаковывается в выбранную папку), **экспорт/импорт всего диска в .zip**.
- **Редактор AUTOEXEC.BAT / CONFIG.SYS** прямо в браузере.
- **Поделиться ссылкой** `?repo=owner/name` (+ QR-код) — публичные диски открываются одним кликом.
- **Индикатор лимита GitHub API**, настройки DOSBox (cycles, звук), сброс локальных данных.

## Запуск локально
```
python3 serve.py        # http://localhost:3000 (заголовки no-cache, удобно для разработки)
# или любой статический сервер: python3 -m http.server 5173 --bind 0.0.0.0
```
Это статический сайт — один `index.html` + `vendor/`. Сборки/Node не требуется.

## Публикация (GitHub Pages)
1. Запушить репозиторий в `github.com/sementsul/GitDOS`.
2. **Settings → Pages → Source: GitHub Actions** (workflow `.github/workflows/deploy-pages.yml` опубликует корень).
3. Сайт: `https://sementsul.github.io/GitDOS/`.

PWA: на опубликованном сайте включается сервис-воркер (`sw.js`, **network-first** — онлайн всегда свежая
версия, кэш только для офлайна). На `localhost` он намеренно отключён, чтобы не мешать разработке.

## Безопасность
GitHub-токен пользователь вводит **в браузере** (хранится в localStorage), в код/репозиторий он **не попадает**
и **не кладётся в ссылки «Поделиться»**. Для приватных репозиториев и записи нужен токен с правом `repo`.

Заметки по разработке и грабли — `docs/project-notes.md`.
